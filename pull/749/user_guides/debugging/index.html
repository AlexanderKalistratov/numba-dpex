Debugging with Intel® Distribution for GDB*

Numba-dppy allows you to debug SYCL* kernels with Intel® Distribution for GDB*. To enable the emission of debug information, set the debug environment variable NUMBA_DPPY_DEBUGINFO, for example: export NUMBA_DPPY_DEBUGINFO=1 To disable debugging, unset the variable: unset NUMBA_DPPY_DEBUGINFO

Note

Enabling debug information significantly increases the memory consumption for each compiled kernel. For a large application, this may cause out-of-memory error.

Not all debugging features supported by Numba on CPUs are yet supported by Numba-dppy. See Features and Limitations.

Requirements

Intel® Distribution for GDB* is required for Numba-dppy debugging features to work. Intel® Distribution for GDB* is part of Intel oneAPI. For relevant documentation, refer to the `Intel® Distribution for GDB* product page`_.
